# סוכן חדשנות בבריאות — מקבץ שבועי

תאום של סוכן "הבריאות והלונג'ביטי", מכוון לששת אתרי חדשנות-הבריאות שאומתה עבורם גישה מלאה לכתבה האחרונה. רץ **אחת לשבוע** (יום ראשון בבוקר), מסכם עם **Opus 4.8**, שולח במייל דרך Gmail, ושומר מצב ב-GitHub.

## ששת האתרים
1. Fierce Healthcare
2. MobiHealthNews
3. MedCity News
4. npj Digital Medicine (Nature)
5. Healthcare IT News
6. Rock Health *(מפרסם מדי כמה שבועות — רוב השבועות פשוט ידולג)*

## מה שונה מהסוכן הקיים (3 שינויים בלבד)
- **רשימת מקורות** — ששת אתרי החדשנות במקום אתרי התזונה/לונג'ביטי.
- **תדירות** — שבועי (הוסר השער הדו-שבועי; כל ריצת יום ראשון שולחת).
- **בחירה** — הכתבה האחרונה מכל אתר (`MAX_PER_SOURCE=1`), עד 6 כתבות.

כל שאר הלוגיקה זהה: dedup מול `seen.json`, משיכת טקסט מלא כשהפיד נותן רק תקציר, ניסוח עברי במילים של קלוד עם קישור למקור, מייל RTL, ושמירת מצב חזרה ל-repo.

## הקמה (חד-פעמי)
> חשוב: פתחו **repo נפרד** (או לפחות `seen.json` נפרד) מהסוכן הקיים, אחרת שני הסוכנים ידרסו זה את המצב של זה.

1. צרו repo חדש בגיטהאב והעלו את הקבצים:
   ```
   digest.py
   requirements.txt
   seen.json
   README.md
   .github/workflows/health-innovation-digest.yml
   ```
2. **Settings → Secrets and variables → Actions → New repository secret** — הגדירו 4 (זהים לסוכן הקיים):
   - `GMAIL_ADDRESS` — כתובת ה-Gmail השולחת
   - `GMAIL_APP_PASSWORD` — App Password של Gmail (לא הסיסמה הרגילה)
   - `RECIPIENT` — לאן לשלוח (אם ריק, נשלח לכתובת השולחת)
   - `ANTHROPIC_API_KEY` — מפתח ה-API
3. **בדיקה מיידית:** לשונית **Actions → Health Innovation Digest → Run workflow**. תוך דקה-שתיים אמור להגיע מייל.
4. אחרי שהבדיקה עבדה — זהו. הסוכן ירוץ אוטומטית כל יום ראשון ב-05:00 UTC (≈08:00 בישראל).

## שלושה מנגנוני הגנה (נוספו אחרי ביקורת קוד)
1. **מכסה קשיחה של כתבה אחת לאתר.** אם רק 4 אתרים פרסמו השבוע — תקבל 4 כתבות מ-4 אתרים, ולא "מילוי" בכתבה שנייה ושלישית מאותו אתר. עדיף מייל קצר ונכון על מייל מלא ומטעה.
2. **ריטריי על שגיאות API.** 429 (rate limit) או 529 (Opus בעומס) לא מפילים את הסיכום לאנגלית בשקט — יש עד 4 ניסיונות עם השהיה מתגברת. שגיאה קבועה (מפתח שגוי) נכשלת מיד במקום לבזבז ניסיונות. אם סיכום בכל זאת נכשל, הכרטיס מסומן במפורש "(תקציר מהמקור; הסיכום האוטומטי נכשל)" כדי שלא תחשוב שזה סיכום תקין.
3. **קו-מים לכל אתר (watermark).** הסוכן לעולם לא ישלח כתבה **ישנה יותר** מזו שכבר נשלחה מאותו אתר. בלי זה, אתר שלא פרסם השבוע (Rock Health) היה שולח לך דוח מלפני חודש, שבוע אחרי שקיבלת את החדש.

## כוונונים (אופציונלי, דרך Secrets/variables)
- `ARTICLES_PER_RUN` (ברירת מחדל 6)
- `MAX_PER_SOURCE` (ברירת מחדל 1 — כתבה אחת לכל אתר)
- `STRICT_PER_SOURCE` (ברירת מחדל true — מכסה קשיחה, בלי מילוי)
- `MAX_AGE_DAYS` (ברירת מחדל 30 — חלון טריות)
- `API_MAX_RETRIES` (ברירת מחדל 4)
- `API_PACING_SECONDS` (ברירת מחדל 2 — השהיה בין קריאות ל-Opus)
- `ANTHROPIC_MODEL` (ברירת מחדל `claude-opus-4-8`)
- `DIGEST_TITLE` (ברירת מחדל "חדשנות בבריאות")

## מבנה seen.json
```json
{ "links": ["..."], "watermarks": {"Fierce Healthcare": "2026-07-10T12:00:00+00:00"}, "updated": "..." }
```
תואם לאחור: קובץ ישן בלי `watermarks` ייקרא כרגיל ויתעדכן אוטומטית בריצה הראשונה.

## הערה על הפידים
כתובות ה-RSS של Fierce, MobiHealthNews ו-MedCity אומתו, ו-npj משתמש בתבנית ה-RSS הסטנדרטית של Nature. אם אחת הכתובות תשתנה בעתיד, הסוכן מזהה זאת אוטומטית ומגלה את הפיד מחדש מדף הבית (אותו מנגנון fallback מהסוכן המקורי), כך שריצה לא נשברת בגלל שינוי כתובת בודד.
